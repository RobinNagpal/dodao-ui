# Research ‚Äî "Add New Report" functionality

Reference notes for anyone who needs to **add a new report** in KoalaGains (insights-ui).
Written because "add a new report" is ambiguous ‚Äî it means two very different things
depending on the surface. This doc disambiguates them and gives a concrete file checklist
for each.

All paths are relative to `insights-ui/` (the Next.js app root) unless noted.

---

## TL;DR ‚Äî two different "add a new report" tasks

| # | What "add a report" means | Who does it | Where |
|---|---------------------------|-------------|-------|
| 1 | **Add a new report _instance_** of an existing type (run analysis for one more stock / ETF / template) | Admin / automation, at runtime, no code change | Admin UI buttons + CLI scripts |
| 2 | **Add a new report _type / category_** (a brand-new kind of analysis, e.g. the way `management-team` was added) | Developer, code change across ~30 files | Enum + prompt + save + page + sitemap wiring |

Pick the section that matches what's actually being asked. Most Discord/product
requests are **#1** ("generate the reports for these tickers"); "add a new report"
in a _spec/roadmap_ sense usually means **#2** or the Custom-Reports feature below.

---

## Report families in the codebase

There are four independent report subsystems. They do **not** share a generation path.

1. **Stock reports (Ticker V1)** ‚Äî 8 report types, enum in
   `src/types/ticker-typesv1.ts` (`ReportType`, lines 138‚Äì160).
   Types: `financial-analysis`, `competition`, `business-and-moat`, `past-performance`,
   `future-growth`, `fair-value`, `management-team`, `final-summary` (must run last).
2. **ETF reports (ETF V1)** ‚Äî 7 report types, enum/consts in
   `src/types/etf/etf-analysis-types.ts`. Mirrors the stock pipeline (category-analysis
   results, competition, key-facts, future-returns, final-summary).
3. **Tariff / industry reports** ‚Äî one big multi-section JSON row per industry chapter.
   Prisma `TariffChapterReport`. Generated by an ordered script sequence
   (`src/scripts/.../00-‚Ä¶` through `08-‚Ä¶`).
4. **Analysis-Template reports** ‚Äî the flexible/"custom" report engine driven by
   `AnalysisTemplate` ‚Üí `AnalysisTemplateCategory` ‚Üí `AnalysisTemplateParameter`, with
   instances in `AnalysisTemplateReport` / `AnalysisTemplateParameterReport`.

## #1 ‚Äî Add a new report _instance_ (no code change)

### Analysis-Template reports ‚Äî the literal "Create New Report" button

This is the only surface with an actual **"Create New Report"** button in the UI.

- **Admin list page:** `src/app/admin-v1/analysis-template-report/page.tsx`
  ("Create New Report" button at line ~100; Edit / Delete / Generate per row).
- **Create/Edit modal:** `src/components/analysis-templates/AddEditAnalysisTemplateReportModal.tsx`
  - Fields: **Analysis Template** (dropdown), **Prompt Key**, **Report Name**, **Input JSON**.
  - Modal title switches on `isEdit`: `Create Analysis Template Report` vs `Edit Analysis Template Report` (line ~120).
  - `POST` creates, `PUT` updates.
- **API:** `src/app/api/analysis-template-reports/route.ts`
  - `GET` (list, line ~89) / `POST` (create, line ~90); request shape
    `CreateAnalysisTemplateReportRequest = { analysisTemplateId, promptKey, reportName, inputObj }` (line 7).
  - Per-report ops: `src/app/api/analysis-template-reports/[analysisTemplateReportId]/route.ts` (`PUT` / `DELETE`).
- **Detail / generate pages:**
  `src/app/admin-v1/analysis-template-report/[analysisTemplateReportId]/page.tsx` and
  `.../[analysisTemplateReportId]/generate/page.tsx`.

### Stock reports ‚Äî "Create Reports" admin page + CLI

- **Admin page:** `src/app/admin-v1/create-reports/page.tsx` ‚Äî pick tickers, pick report
  types (all or a subset), enqueue generation. Progress tracked on
  `src/app/admin-v1/generation-requests/page.tsx`.
- **Generation API:** `POST src/app/api/[spaceId]/tickers-v1/generation-requests/route.ts`
  (enqueues a `TickerV1GenerationRequest` per selected type).
- **CLI runbook:** `docs/insights-ui/stock-analysis/generate-stock-reports.md`
  (`yarn stocks:trigger` / `yarn stocks:save` / `yarn stocks:list-oldest`).
  Prereq: the stock must exist ‚Äî `yarn stocks:add` (`docs/insights-ui/stock-analysis/add-stock.md`).

### ETF reports ‚Äî parallel to stocks

- **Admin pages:** `src/app/admin-v1/etf-reports/page.tsx`,
  `src/app/admin-v1/etf-generation-requests/page.tsx`.
- **Generation API:** `POST src/app/api/[spaceId]/etfs-v1/generation-requests/route.ts`
  (extra pre-check: `ensure-mor-info` refreshes the data-provider snapshot before generating).
- **CLI runbook:** `docs/insights-ui/etf-analysis/generate-etf-reports.md`.

### Tariff reports

- **Admin page:** `src/app/admin-v1/tariff-reports/page.tsx`.
- **Generation:** ordered scripts `src/scripts/.../00-industry-main-headings.ts` ‚Ä¶
  `08-report-seo-info.ts`, plus `POST /api/industry-tariff-reports/[industry]/generate-*` routes.

### End-to-end automation reference

`docs/insights-ui/automated-report-generation.md` documents the agent-driven
"get prompt ‚Üí act as the LLM ‚Üí save" loop for stocks and ETFs.

## #2 ‚Äî Add a new report _type_ (developer task)

A new stock report category (like `management-team`, the most recently added one) touches
~30 files. Grep `management-team` / `MANAGEMENT_TEAM` for the canonical worked example.
The wiring falls into these buckets:

### a) Type / enum definition
- `src/types/ticker-typesv1.ts` ‚Äî add to `ReportType` enum **and** `analysisTypes[]` label list.
- `src/types/public-equity/analysis-factors-types.ts` ‚Äî factor/schema types for the new report.

### b) Prompt generation
- `src/utils/analysis-reports/prompt-generator-utils.ts` ‚Äî build the prompt for the new type.
- `src/scripts/tickers/get-report-prompt.ts` ‚Äî CLI hook.
- Prompt source-of-truth text lives under `docs/insights-ui/stock-prompts/`
  (e.g. `management-team`) ‚Äî add a new folder for the new type.

### c) Persistence (save)
- `src/utils/analysis-reports/save-report-utils.ts` + `save-report-callback-utils.ts` ‚Äî validate & write.
- `src/scripts/tickers/save-report.ts` ‚Äî CLI save hook.
- **Prisma:** if the report needs its own table (management-team added `TickerV1ManagementTeamReport`),
  add the model + migration; otherwise it reuses `TickerV1CategoryAnalysisResult`.
- Save endpoint: `src/app/api/[spaceId]/tickers-v1/exchange/[exchange]/[ticker]/save-json-report/route.ts`
  (+ a dedicated read route like `.../[ticker]/management-team/route.ts` if it has its own table).

### d) Generation status / requests
- `src/utils/analysis-reports/report-steps-statuses.ts`, `report-status-utils.ts`,
  `report-generator-utils.ts`, `generation-report-utils.ts`.
- `src/scripts/tickers/trigger-generation.ts`, `src/hooks/useGenerateReports.ts`.
- Request routes: `src/app/api/[spaceId]/tickers-v1/generation-requests/route.ts`,
  `.../[ticker]/generation-requests/route.ts`, `.../[ticker]/creation-infos/route.ts`.

### e) Rendering (public detail page)
- `src/app/stocks/[exchange]/[ticker]/page.tsx` (main detail page includes the section).
- `src/app/stocks/[exchange]/[ticker]/<report-type>/page.tsx` ‚Äî dedicated sub-page.
- `src/components/ticker-reportsv1/TickerRelatedSections.tsx`,
  `src/components/ui/sections/ReportArticleShell.tsx`, `ReportFooter.tsx`.
- Debug view: `src/components/ticker/debug/Debug<Report>.tsx`,
  `src/app/public-equities/debug/tickers/[ticker]/TickerDetailsDebugPage.tsx`.

### f) SEO / sitemap / cache
- `src/app/stocks/<report-type>-sitemap.xml/route.ts` ‚Äî new per-report sitemap.
- `src/utils/metadata-generators.ts`, `src/utils/ticker-v1-cache-utils.ts`,
  `src/utils/ticker-v1-model-utils.ts`.
- `src/app/admin-v1/invalidate-cache/page.tsx` ‚Äî expose the new type for cache busting.

> ETF side is the same buckets under `src/**/etfs-v1/**`,
> `src/utils/etf-analysis-reports/**`, `src/types/etf/**`.

## Related roadmap item ‚Äî Custom Reports ("random reports") per stock

There is already an open spec in [`todo-tasks.md`](./todo-tasks.md) ‚Üí **Stocks ‚Üí Custom
Reports ("random reports") per stock** (source: PR #1318). That is a _user-facing_,
per-ticker, free-form/ templated report feature (`TickerV1CustomReport` +
`TickerV1CustomReportRun`, a `[+ New Report]` button on the stock detail page, synchronous
LLM run via `getLLMResponseForPromptViaInvocation`). If the ask behind "add new report" is
letting users create arbitrary reports, **that spec is the plan ‚Äî don't duplicate it here.**
This doc is the architectural map; that task is the build.

## Auth / access notes

- Stock & ETF generation endpoints are gated by `withAdminOrToken`
  (admin JWT **or** `x-automation-token`); CLI scripts need `AUTOMATION_SECRET`.
- Admin pages under `/admin-v1` require JWT login.
