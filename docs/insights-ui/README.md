# Insights-UI Knowledge

Topical reference docs for the Insights-UI (KoalaGains) app вЂ” patterns, prompts, runbooks, analysis methodology, plus the active task lists that drive ongoing work.

## Subfolders

- **[etf-analysis/](etf-analysis/)** вЂ” How the ETF analysis pipeline works end-to-end: report generation and the ETF Scenarios system (winner / loser model + admin/automation paths). See [etf-analysis/README.md](etf-analysis/README.md).
- **[stock-analysis/](stock-analysis/)** вЂ” Runbooks for adding new stocks (`yarn stocks:add`) and triggering stock report generation (`yarn stocks:trigger`). See [stock-analysis/README.md](stock-analysis/README.md).
- **[etf-prompts/](etf-prompts/)** вЂ” Source-of-truth prompt text for each ETF analysis category (past returns, cost & efficiency, risk, future outlook, intro/strategy, competition) plus the prompt-finalization approach. See [etf-prompts/README.md](etf-prompts/README.md).
- **[scenario-prompts/](scenario-prompts/)** вЂ” Reusable prompt templates for stock & ETF scenario authoring. Currently: `detailed-analysis.md` for generating the optional `detailedAnalysis` long-form section.
- **[stock-prompts/](stock-prompts/)** вЂ” Source-of-truth prompt text for stock analysis report types (currently `management-team`). The prompts themselves live in the `prompt_versions` DB table; the files here are the human-readable, version-controlled copy used for review and seeding. See [stock-prompts/README.md](stock-prompts/README.md).
- **[etf-prompt-improvement/](etf-prompt-improvement/)** вЂ” Methodology + runbook for the prompt-tuning loop (`etf-verification-loop.md`, `run-prompt-analysis.md`) plus iterative prompt-review notes, factor-set reviews, and per-ETF audits. See [etf-prompt-improvement/README.md](etf-prompt-improvement/README.md).
- **[tariffs/](tariffs/)** вЂ” Reference docs for the tariffs subsystem: pipeline, data structures, UI components, S3 storage, admin flow, and the use-case catalog that drives feature prioritization.
- **[tasks/](tasks/)** вЂ” Active KoalaGains task lists (open + closed work, per surface): ETFs, stocks, tariffs, scenarios, prompt tuning, plus open questions and per-page checklists.

## Top-level files

- **[automated-report-generation.md](automated-report-generation.md)** вЂ” Five CLI scripts (`stocks:prompt`, `stocks:save`, `stocks:list-oldest`, `etfs:prompt`, `etfs:save`) that let Claude drive the full 7-ETF / 8-stock report-generation loop, plus a "refresh the N stocks with the oldest reports" batch flow: setup, report order, per-type response JSON shapes, run-time gotchas (JSON-escape quotes, partial MOR data on first prompt, `final-summary` must be last), and the end-to-end copy-paste skeleton.
- **[scenario-authoring.md](scenario-authoring.md)** вЂ” How to draft a new stock or ETF market scenario: scratch-file workflow under `/tmp/scenarios/{stocks,etfs}/<slug>.md`, the required content template (dates, magnitude, per-industry impact numbers, asymmetric winners / losers), per-stock bullet syntax, the outlook phrasing the parser keys off, and the editorial "five winners, five losers" convention.
- **[stock-page-caching.md](stock-page-caching.md)** вЂ” Reference for the ISR / Data Cache topology under `/stocks/[exchange]/[ticker]` and its subpages: per-route tag map, every code path that invalidates a `ticker_*` tag, and the structural reason ISR writes still exceed reads after PRs #1472/#1473 (each rebuild costs one HTML entry + one Data Cache entry per tagged `fetch()` on the page).
- **[aws-migration-plan.md](aws-migration-plan.md)** вЂ” Plan to move `insights-ui` off Vercel onto a Terraform-managed AWS deployment on a **single-node Lightsail Container Service**, rolled out in parallel with Vercel: Phase A runs the app directly at **`prod.koalagains.com`** (no CloudFront), static served from **S3** via `assetPrefix`, DB on the **existing public RDS**, **crons moved to EventBridge** (removed from `vercel.json`); Phase B puts CloudFront in front; Phase C retires Vercel. Covers the required app/code changes (`output: 'standalone'`, `assetPrefix`, `VERCEL_ENV` auth coupling, `waitUntil`в†’`after()`, `maxDuration=300` async refactor, single-node cache coherence), CI/CD, deploy-skew, runbook, rollback, and cost. The Terraform + Dockerfile that implement it live in [`deployments/insights-ui/`](../../deployments/insights-ui/).
- **[cloudfront-deploy-skew.md](cloudfront-deploy-skew.md)** вЂ” Full caching architecture for `/stocks/*` and `/etfs/*` on koalagains.com: the three-phase journey from `force-static` + ISR (too expensive due to ISR writes flushed on every deploy), through adding CloudFront in front of Vercel (introduced a deploy-skew "Something went wrong" bug), to today's `force-dynamic` + CloudFront + Vercel Skew Protection setup. Covers the deploy-skew fix in detail, where CloudFront is still slow (cold-cache misses, long-tail pages, RSC navigation), the constraints we operate under (no deploy invalidations, zero ISR writes, low function invocations), and the three options for the next iteration (RSC caching via CloudFront Function, selective Cache-Control via Next.js middleware, or status quo).
